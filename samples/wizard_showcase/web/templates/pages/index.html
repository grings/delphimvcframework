Minimal API WebApp Showcase

Five endpoints, each demonstrating a different parameter-binding mode in a server-rendered context.

GET /people
DI interface

Lambda receives IPeopleService resolved from the service container.
GET /people/(id:int)
primitive route segment

Integer arg bound from the next unconsumed route segment.
DELETE /people/(id:int)
MapDelete + HTMX hx-delete

Wired to the per-row Delete button on /people. Returns empty body; HTMX swaps the <tr> out.

GET /search
[MVCFromQueryString] + defaults

Record bound from query-string params with per-field default values. Omit any param — the default kicks in.
GET / POST /signup
[MVCFromContentField] + TArray<string>

Record bound from form-urlencoded fields, including a multi-value checkbox group that lands in a TArray<string>.
GET /context
[MVCFromHeader] + [MVCFromCookie]

Record bound from request headers and cookies, with per-field default values when the source is missing.

All routes are mounted via ARoot.AsWeb; /people/* additionally goes through a logging endpoint filter attached with .Use(LogFilter()). Read RoutesU.pas and ShowcaseModelsU.pas — both files are heavily commented.
